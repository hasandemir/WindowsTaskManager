        If a process matching match has metric op threshold for for seconds, run action.

        OnNameMatchMetricOpThresholdFor (s)ActionCooldown (s)
        Help & examples
        • metric=memory_bytes op=>= threshold=4000000000 for=30 — kill if process uses ≥ 4 GB for 30s
        • metric=cpu_percent op=>= threshold=90 for=60 action=suspend — suspend runaway at 90% for 60s
        • metric=thread_count op=>= threshold=500 — alert on thread leaks
        • Match is a case-insensitive substring of the process name (e.g. chrome.exe, node).
        • Cooldown defaults to 60s — prevents repeated kills of restarted processes.

          Windows Task Manager
          
      
          A pure-Go, single-binary replacement for the built-in Windows Task Manager — with a live web dashboard, REST API, anomaly engine, and an LLM advisor that proposes fixes you approve one click at a time.
          

      
          What's in the box
          
      
            
        
          • Live metrics. CPU (per-core), memory, GPU, disks, network, processes, process tree, TCP/UDP endpoints — all over Server-Sent Events.
            • 
        
          • Process control. Kill, suspend, resume, priority, affinity, Job-Object CPU/memory caps. A protection list keeps critical processes safe from accidental clicks.
            • 
        
          • Per-process toggles. One click in the process table to protect (🛡) or ignore (🔕) any executable. Both lists persist to config.yaml.
            • 
        
          • Sortable table. Click any column header to sort; click again to flip direction.
            • 
        
          • Anomaly detection — conservative by default. Eight detectors ship; only runaway-CPU, memory-leak and spawn-storm fire out of the box. The noisier ones (hung, orphan, port-conflict, network burst) are opt-in.
            • 
        
          • Automation rules. "If chrome.exe uses ≥ 4 GB for 30 s, kill it." Hot-reloaded from YAML or edited live in the Rules tab.
            • 
        
          • AI advisor with approve-before-execute. Works with Anthropic Claude or any OpenAI-compatible provider (OpenAI, OpenRouter, Groq, DeepSeek, Ollama, LM Studio, …). The advisor emits structured action suggestions; nothing runs until you click Approve.
            • 
        
          • System tray. Native NotifyIcon, rate-limited balloon notifications, right-click menu.
            • 
      
          

      
          Tech
          
      
          Pure Go — no CGo, no WMI, no PowerShell. Direct Win32 syscalls via golang.org/x/sys/windows. The web UI is vanilla JS, embedded into the binary with embed.FS. Ships as a single wtm.exe under ~12 MB.
